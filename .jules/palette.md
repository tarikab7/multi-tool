## 2024-06-14 - Interactive Spans and Icon-only Dynamic Buttons
**Learning:** In custom interactive elements like theme selectors built with `span` tags, relying solely on CSS for active states and click handlers leaves screen reader and keyboard users stranded. Similarly, dynamically rendered icon-only buttons (like the favorites star) need dynamic updates to their ARIA labels and pressed states alongside their visual updates.
**Action:** When auditing or implementing interactive UI elements that aren't native `<button>` or `<input>`, explicitly assign `role="button"`, add `tabindex="0"`, bind keyboard listeners for `Enter` and `Space`, and map internal states to `aria-pressed`. For icon-only buttons with toggling states, update both `aria-label` and `aria-pressed` within the state change logic.
